# HesabBan (حساب‌بان) — Prototype v1

اپلیکیشن اندروید شخصی برای خوندن پیامک‌های بانکی، پارس کردن مبلغ/نوع تراکنش،
ثبت دستی و گزارش هفتگی/ماهانه. همه‌چیز آفلاین و روی خود گوشی ذخیره می‌شه (Room DB).

این نسخه مطابق مشخصات موجود در `design_handoff_hesabban_android/` (همون پوشه‌ای
که قبلاً در همین workspace ساخته شده بود: README.md / ANDROID_GUIDE.md /
SMS_PARSING.md) پیاده‌سازی شده — دسته‌بندی‌ها، منطق پارس پیامک، allow-list
شماره‌های بانک، و مدل داده از همون سه فایل گرفته شده.

## چیزی که هست
- **Allow-list شماره فرستنده**: فقط پیامک شماره‌هایی که در تب «تنظیمات» فعال کردی پردازش می‌شن.
- خوندن پیامک لحظه‌ای (BroadcastReceiver) + وارد کردن تاریخچه پیامک‌های قبلی (فقط از شماره‌های مجاز)
- پارسر مطابق `SMS_PARSING.md`: نرمال‌سازی ارقام فارسی/عربی، کلمات کلیدی واریز/برداشت،
  استخراج مبلغ + مانده حساب، تشخیص ریال/تومان، 4 رقم آخر کارت، حذف پیامک‌های رمز/OTP
- Dedupe با sha1(sender+body) — یک پیامک دوبار ثبت نمی‌شه
- همگام‌سازی موجودی حساب از روی «مانده/موجودی» داخل پیامک
- دسته‌بندی‌ها دقیقاً طبق spec: خوراک، حمل‌ونقل، قبض‌ها، خرید، درآمد، سایر
- افزودن/حذف دستی تراکنش با انتخاب دسته از همون ۶ دسته
- گزارش هفتگی و ماهانه به تفکیک دسته‌بندی
- دیتابیس Room محلی، بدون سرور، بدون اینترنت — هیچ پرمیشن INTERNET نداره

## چیزی که نیست (هنوز) — صادقانه بگم کجا کوتاه اومدم
- **ظاهر «Modernist» پیکسل‌به‌پیکسل** (زاویه صفر، فونت Vazirmatn، RTL کامل،
  toggleها و chipهای سفارشی) پیاده نشده. UI فعلی Material3 پیش‌فرضه —
  فقط برای تست منطق کار می‌کنه، شبیه mockup نیست. این خودش یه کار جدا و نسبتاً بزرگه.
- **override map برای دسته‌بندی خودکار** (وقتی یه بار دستی دسته یه merchant رو عوض کنی،
  دفعه بعد خودش همون دسته رو بده) پیاده نشده؛ فعلاً فقط همون تراکنش تکی ویرایش می‌شه.
- **تاریخ جلالی** (امروز/دیروز/`۱۸ تیر`) پیاده نشده؛ فعلاً تاریخ میلادی ساده نمایش داده می‌شه.
- **Onboarding سه‌مرحله‌ای** و **نوتیفیکیشن** بعد از ثبت خودکار تراکنش پیاده نشده.
- ویرایش تراکنش موجود (فقط افزودن/حذف دستی فعلاً؛ نه edit).

## راه‌اندازی
1. Android Studio (نسخه Hedgehog به بعد) رو باز کن.
2. پوشه `HesabBan` رو به عنوان پروژه Open کن — Gradle خودش sync می‌شه (نیاز به JDK 17).
3. روی گوشی یا امولاتور (minSdk 26+) اجرا کن.
4. اولین بار که باز میشه، اجازه READ_SMS / RECEIVE_SMS رو بده.
5. برو تب «تنظیمات» و شماره فرستنده پیامک بانکت رو اضافه کن (اسم بانک + شماره) — **بدون این مرحله هیچ پیامکی پردازش نمی‌شه**، این عمداً همینطوریه (allow-list).
6. دکمه «وارد کردن از پیامک» تاریخچه ۳ ماه اخیر رو از شماره‌های فعال اسکن و پارس می‌کنه.

## نکته مهم درباره پارسر [محتمل]
فرمت پیامک بانک‌ها متفاوته (ملی، سامان، ملت، سپه...). پارسر فعلی قوانین
`SMS_PARSING.md` رو پیاده کرده ولی روی پیامک واقعی تست نشده — این بخش رو باید
با نمونه پیامک واقعی (بدون اطلاعات حساس) تست کنی و کلمات کلیدی رو در
`parser/SmsParser.kt` تنظیم کنی.

## من نمی‌تونستم اینجا build/کامپایل کنم [قطعی]
محیطی که این کد توش نوشته شده SDK اندروید نداره و دسترسی root/apt هم نداره، و
شبکه‌اش برای دانلود کامل Android SDK + Gradle + وابستگی‌ها خیلی کند بود (تست
کردم، عملی نبود). کد رو خط‌به‌خط بازبینی کردم ولی اولین بار که واقعاً build
بشه، احتمال داره چندتا خطای کوچیک sync/import پیش بیاد که باید دستی رفع بشه.

## گرفتن APK بدون نصب Android Studio — با GitHub Actions
این پروژه یک workflow آماده داره (`.github/workflows/build-apk.yml`) که خودش
روی سرورهای GitHub، اپ رو build می‌کنه و یک فایل APK قابل نصب تحویل می‌ده.
مراحل (نیازی به نصب چیزی روی کامپیوتر نیست، فقط مرورگر):

1. یک حساب رایگان در github.com بساز (اگه نداری).
2. یک repository جدید بساز (دکمه سبز "New") — اسمش مهم نیست، مثلاً `hesabban`. می‌تونه Private باشه.
3. تو صفحه‌ی همون repository خالی، لینک "uploading an existing file" رو بزن،
   و کل پوشه‌ی استخراج‌شده‌ی این zip رو (با زیرپوشه‌هاش) بکش و رها کن (drag & drop) —
   مرورگرهای مدرن ساختار پوشه‌ها رو حفظ می‌کنن. بعد "Commit changes" رو بزن.
4. برو به تب **Actions** بالای صفحه‌ی repository. یک workflow به اسم
   "Build HesabBan APK" باید خودش شروع به اجرا شده باشه (چند دقیقه طول می‌کشه).
5. وقتی علامت ✅ سبز کنارش اومد، روش کلیک کن؛ پایین صفحه بخش **Artifacts**
   یک فایل به اسم `hesabban-debug-apk` هست — دانلودش کن (یک zip میاد که توش
   `app-debug.apk` هست).
6. اون APK رو به گوشیت منتقل کن (ایمیل به خودت، گوگل درایو، یا کابل USB).
7. رو گوشی: تنظیمات → امنیت → «نصب اپ‌های ناشناس» رو برای اپی که فایل رو باهاش
   باز می‌کنی (مثلاً Files یا Chrome) فعال کن.
8. روی فایل APK بزن → Install.
9. اپ رو باز کن، دسترسی SMS رو بده، و از تب «تنظیمات» شماره پیامک بانکت رو اضافه کن.

اگه workflow قرمز (❌) شد یعنی یه جای کد خطای کامپایل داره — روی run کلیک کن،
لاگ خطا رو کپی کن و برام بفرست تا رفعش کنم.

نکته امنیتی [محتمل]: این یک debug APK ه (با کلید امضای پیش‌فرض دیباگ)، برای
استفاده‌ی شخصی خودت کاملاً کافیه؛ فقط برای انتشار عمومی در Google Play مناسب نیست.
